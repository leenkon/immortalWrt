# ImmortalWrt 主旁路由配置教程

## 网络拓扑

```
光猫 (桥接模式)
    │
    │ PPPoE 拨号
    │
    ▼
┌─────────────────┐      ┌─────────────────┐
│   主路由 (main)  │      │  旁路由 (bypass) │
│   10.10.10.1    │◄────►│   10.10.10.99   │
│                 │ LAN/WAN│               │
└─────────────────┘      └─────────────────┘
```

## IP 地址规划

| 设备 | IP 地址 | 说明 |
|------|---------|------|
| 主路由 | 10.10.10.1 | 默认不变 |
| 旁路由 | 10.10.10.99 | 默认不变 |
| DHCP 范围 | 10.10.10.100 - 10.10.10.250 | 由主路由分配 |

## 一、主路由配置 (default-main)

### 1.1 基础设置

| 设置项 | 值 |
|--------|-----|
| 主机名 | Router-Main |
| IP 地址 | 10.10.10.1 |
| 时区 | Asia/Shanghai (CST-8) |

### 1.2 PPPoE 拨号

| 设置项 | 值 |
|--------|-----|
| 协议 | PPPoE |
| 账号 | 您的宽带账号 |
| 密码 | 您的宽带密码 |

### 1.3 DHCP 服务器（关键设置）

| 设置项 | 值 |
|--------|-----|
| 状态 | 开启 |
| 开始地址 | 10.10.10.100 |
| 结束地址 | 10.10.10.250 |
| 子网掩码 | 255.255.255.0 |
| DHCP 租期 | 12 小时 |
| **DNS 服务器** | **10.10.10.99, 223.5.5.5** |

### 1.4 防火墙

- 保持默认设置
- 开启 Full-NAT（已内置）

### 1.5 其他功能

| 功能 | 状态 | 说明 |
|------|------|------|
| DDNS | 可选 | 动态域名 |
| UPnP | 开启 | 端口映射 |
| VLAN | 可选 | 交换机配置 |
| IPTV | 可选 | 观看 IPTV |
| 多拨 (mwan3) | 可选 | 负载均衡 |
| OAF | 可选 | 应用过滤（主路由和旁路由均可选） |

## 二、旁路由配置 (mini-bypass)

| 设置项 | 值 |
|--------|-----|

| IP 地址 | 10.10.10.99 |
| 时区 | Asia/Shanghai (CST-8) |

### 2.2 DHCP 服务器

| 设置项 | 值 |
|--------|-----|

### 2.6 广告过滤配置

| 设置项 | 值 |
|--------|-----|
| 状态 | 启用 |

## 三、客户端设备配置（推荐方案）

### 3.1 所有设备统一配置

| 设置项 | 值 |
|--------|-----|
| IP 获取 | DHCP 自动获取 |
| DNS | 自动（主路由分配：10.10.10.99, 223.5.5.5） |
| 网关 | 自动 |

**优势：**
- ✅ 所有设备自动享受广告过滤
- ✅ 旁路由故障时，备用 DNS 223.5.5.5 自动接管
- ✅ 无需单独配置每台设备

---

## 四、流量走向

```
普通流量：
客户端 → 主路由 DHCP → 主路由网关 → 光猫 → 互联网

广告过滤：
客户端 DNS 查询 → 旁路由 (adblock) → 过滤 → 主路由 → 互联网

科学上网：
客户端 → 旁路由 (OpenClash 劫持) → 代理服务器 → 目标网站

故障切换：
客户端 DNS 查询 → 旁路由故障 → 自动切换到 223.5.5.5 → 互联网
```

---

## 五、故障排查

### 5.1 旁路由无法上网

1. 检查旁路由网关是否指向 10.10.10.1
2. 检查 DNS 是否可用的（223.5.5.5）
3. 检查主路由是否正常工作

### 5.2 OpenClash 不生效

1. 检查客户端 DNS 是否指向 10.10.10.99
2. 检查 OpenClash 是否正常运行
3. 检查订阅链接是否有效

### 5.3 广告过滤无效

1. 检查 DNS 是否指向旁路由
2. 检查广告过滤是否启用
3. 检查过滤列表是否更新

### 5.4 旁路由故障不影响上网

- 主路由 DHCP DNS 设置为 `10.10.10.99, 223.5.5.5`
- 当旁路由故障时，客户端自动使用备用 DNS 223.5.5.5

## 六、快速检查清单

### 主路由检查项

- [ ] PPPoE 拨号成功
- [ ] DHCP 服务器开启
- [ ] DHCP 范围：10.10.10.100 - 10.10.10.250
- [ ] **DHCP DNS：10.10.10.99, 223.5.5.5**
- [ ] LAN IP：10.10.10.1

### 旁路由检查项

- [ ] IP 地址：10.10.10.99
- [ ] DHCP 服务器：**已关闭**
- [ ] 网关：10.10.10.1
- [ ] DNS：10.10.10.1 或 223.5.5.5
- [ ] OpenClash：运行中
- [ ] 广告过滤：已启用
