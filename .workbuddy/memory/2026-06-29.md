## AdGuardHome YAML 结构修复

官方文档（https://github.com/AdguardTeam/AdGuardHome/wiki/Configuration）确认：

**核心错误**：`filters`/`whitelist_filters`/`user_rules` 是**顶层独立配置项**，不能嵌套在 `filtering:` 下。

旧 YAML 结构（错误）：
```yaml
dns:
  protection_enabled: true      # 错误：应在 filtering 下
  filtering_enabled: true       # 错误：应在 filtering 下
  blocked_services: ...         # 错误：应在 filtering 下
filtering:
  filtering_enabled: true
  filters: [...]                # 错误：filters 是顶层项，不应嵌套
  whitelist_filters: []         # 错误：顶层项
  user_rules: []                # 错误：顶层项
```

新 YAML 结构（正确）：
```yaml
dns:                             # 只含 DNS 协议相关配置
  port/bind_hosts/upstream/bootstraps/cache
filtering:                       # 过滤保护相关配置
  protection_enabled/filtering_enabled/blocked_services/blocked_response_ttl
filters:                         # 顶层：过滤器列表
whitelist_filters: []            # 顶层：白名单
user_rules: []                   # 顶层：自定义规则
```

修复内容：
1. `protection_enabled`/`filtering_enabled`/`blocked_services`/`parental_enabled`/`safebrowsing_enabled` 从 `dns:` 移到 `filtering:`
2. `filters` 从 `filtering.filters` 提升为顶层 `filters`
3. `whitelist_filters`/`user_rules` 提升为顶层项
4. filter 条目字段顺序按官方文档调整（enabled → url → name → id）
5. `filtering` 补充缺失字段：`blocking_mode`/`blocked_response_ttl`/`cache_time`/`filters_update_interval`
