# 2026-06-29 工作日志

## 项目代码审查与修正

发现并修复 5 个问题：

1. **diy.sh full 分支 ADGH UCI 配置缺失**（关键 bug）
   - full 分支只设了 `adguardhome.config.enabled='1'`，缺少 `port='5335'` 和 `redirect='0'`
   - UCI init 脚本可能用默认端口 53 覆盖 YAML 的 5335，导致 DNS 链路断裂
   - 修复：补充 `uci set adguardhome.config.port='5335'` 和 `redirect='0'`

2. **upgrade-adgh.sh 版本提取逻辑错误**
   - `cut -d: -f3` 对 `PKG_VERSION:=value` 格式无效（只有2个冒号分隔字段，f3 为空）
   - 导致 CURRENT_VER 恒为空，"已是最新版本"跳过检查永远不生效
   - 修复：改为 `sed -n 's/^PKG_VERSION:=//p'`

3. **upgrade-adgh.sh 缺少 FRONTEND_PKG_VERSION patch**
   - 若 Makefile 存在 `FRONTEND_PKG_VERSION:=` 硬编码行，未随版本升级更新
   - 修复：添加 `sed -i` patch（若变量不存在则 sed 不匹配，安全无副作用）

4. **upgrade-openclash-core.sh 过时注释 + tar -o 冗余参数**
   - 注释"仅旁路由构建时调用"→ 改为"仅旁路由和完整路由构建时调用"
   - `tar zxvfo` 的 `-o` 在解压时无意义（GNU tar 解压忽略 -o），改为 `tar zxf`

5. **workflow 文件名含空格**
   - `ImmortalWrtBuilder_ x86_64.yml` → `ImmortalWrtBuilder_x86_64.yml`（git mv 重命名）

7. **adguardhome-full.yaml 从 configs/ 移到 files/etc/adguardhome/**
   - 统一管理：旁路由版 `adguardhome.yaml` + 完整路由版 `adguardhome-full.yaml` 同目录共存
   - diy.sh full 分支 cp 路径从 `configs/adguardhome-full.yaml` 改为 `files/etc/adguardhome/adguardhome-full.yaml`
   - build.sh `fix_line_endings` 补充覆盖 `adguardhome-full.yaml`

   - dns-hijack / update_aliyun_com.sh / adguardhome.yaml 在 Windows 上可能有 CRLF
   - 路由器 ash 不兼容 CRLF，可能导致脚本执行失败
   - 修复：扩展 fix_line_endings 覆盖 files/ 下的关键文件

## 第二轮审查 - 逻辑梳理与架构修正

发现并修复 3 个问题：

1. **diy.sh 文件清理在源树上操作 → 本地编译后源树被破坏**（架构问题）
   - diy.sh 在 `$PROJECT_ROOT/files/` 上执行 `rm`/`cp`，本地多次编译会导致源树残缺
   - workflow 不受影响（每次 fresh checkout），但本地编译后需 `git checkout files/` 恢复
   - 修复：将文件清理逻辑从 diy.sh 移至 build.sh/workflow，在 openwrt 副本上操作
   - diy.sh after 阶段现在只生成 gitignored 文件（99-custom.sh、shadow），源树保持完整

2. **dnsmasq server list 模式不统一**（健壮性问题）
   - main/full 分支只有第一条 server 用 `del_list + add_list`，其余直接 `add_list`
   - 若默认 config 或手动配置已有 server 条目，可能导致重复
   - 修复：统一改为 `uci delete dhcp.@dnsmasq[0].server` 整体重建 + `add_list`
   - 与 network.lan.dns / network.wan.dns 的 `delete + add_list` 模式一致

3. **workflow cp 语法不一致**（规范性问题）
   - workflow 用 `cp -rf "$GITHUB_WORKSPACE/files/" .`（源带尾斜杠）
   - build.sh 用 `cp -rf "$SCRIPT_DIR/files" "$OPENWRT_DIR/"`（不带尾斜杠）
   - 尾斜杠在 GNU cp 上语义可能不同（内容复制 vs 目录复制）
   - 修复：统一去除尾斜杠，语义更清晰（复制目录本身到目标目录下）
