# 2026-06-28 工作日志

## AdGuardHome 完整修复（多轮排查）

### 1. YAML schema 错误（已修复）
- 根因：`bootstrap_dns` 的 `- 1.1.1.1:53` 未引号包裹，YAML 解析为 mapping 而非 string
- 多次试错：`filter_lists`→`filters`、`bootstrap_dns`→`bootstraps`、duration 字段需带单位、`blocked_services` 格式、缺 `schema_version: 34` 导致 migrator 反复升级失败
- 最终修复：查 AdGuardHome v0.107.76 源码确认正确 schema，写出完整 v34 配置

### 2. 配置文件路径错误（已修复）
- 旧路径：`files/etc/AdGuardHome/AdGuardHome.yaml`（大写）
- 正确路径：`files/etc/adguardhome/adguardhome.yaml`（小写）
- OpenWrt adguardhome 包 uci 默认 `config_file='/etc/adguardhome/adguardhome.yaml'`

### 3. DNS 劫持脚本语法错误（已修复 diy.sh）
- `nft add chain` 的 `{}` 未加引号 → shell 解析报 `unexpected "}"` → 加单引号 `'{ type nat hook prerouting priority -100; }'`
- `firewall include reload='1'` → fw4 不支持 → 删除
- `counter` 关键字位置导致 nft 规则加载失败 → 删除

### 4. AdGuardHome 上游 DNS 超时（已修复）
- 症状：AdGuardHome 查询 1.1.1.1:53 / 223.5.5.5:53 全部 i/o timeout，但 ping 正常
- 根因：旁路由 OpenClash 劫持了出站 UDP 53，AdGuardHome 上游查询被拦截
- 修复：AdGuardHome upstream_dns 改用 DoT（DNS over TLS，端口 853）
  - `tls://1.1.1.1` 和 `tls://223.5.5.5`，绕过端口 53 劫持

### 5. 客户端 DNS 未走旁路由（已修复）
- 根因：主路由无 DNS 劫持规则，手动设 DNS 的客户端绕过 AdGuardHome
- 修复：diy.sh 主路由分支生成 `/etc/dns-hijack.sh`，nftables redirect :53 到本地 dnsmasq，dnsmasq server=10.10.10.2 转发到 AdGuardHome
- 排除 `saddr 10.10.10.2` 避免旁路由自身回环

## 网络设置审查与修正（第2轮）

### 审查结论
- ip_forward：主路由由 fw4 自动启用，旁路由 sysctl.conf 显式写入 — 均正确
- DNS 链路：DHCP 下发 → dnsmasq strictorder+fallback → AdGuardHome DoT — 完整
- 53 端口劫持：nftables prerouting redirect，IPv4 排除旁路由 — 正确

### 修正项
1. **删除主路由 WAN_FW.forward='ACCEPT'**：WAN zone forward 默认 DROP 是正确安全姿态，改为 ACCEPT 无实际需求；DDNS 走 output 链不受影响
2. **添加 IPv6 DNS 劫持规则**：原规则仅覆盖 IPv4（`ip saddr`），IPv6 DNS 可绕过 AdGuardHome；新增 `ip6 nexthdr udp/tcp dport 53 redirect to :53`；IPv6 不排除旁路由（AdGuardHome 上游走 DoT:853，不经 53，无死循环风险）
3. **补上 `firewall.dns_hijack_include.enabled='1'`**：显式启用 firewall include
