## 全面检查主旁路由逻辑，修复 4 个 Bug

### Bug 1 — `uci commit network dhcp` 写法错误（两处）
`uci commit network dhcp` 在 OpenWrt UCI 实现中只 commit 第一个参数 `network`，`dhcp` 被忽略。
修复：拆为 `uci commit network` 和 `uci commit dhcp` 两行分别提交，旁路由和主路由块各修一处。

### Bug 2 — `querytimeout='2000'` 单位错误
dnsmasq UCI 的 `querytimeout` 单位是**秒**，2000 = 2000秒；应为 `'2'`（2秒超时）。已修正。

### Bug 3 — firewall zone 索引 `@zone[lan]`/`@zone[wan]` 不稳定
硬编码下标依赖 zone 在配置文件中的顺序，顺序不同会打错 zone。
改为运行时用 `uci show firewall | grep "\.name='lan'"` 动态查找 zone 路径，赋值给 `LAN_FW`/`WAN_FW` 变量再操作。主路由和旁路由块均已更新。

### Bug 4 — 旁路由 WAN zone 未清干净
原只有 `network=''`，缺少 `masq='0'`，导致 WAN zone 可能残留旧的 masquerade 配置。已补全。

## DHCP DNS 策略最终确定：只下发主路由 IP

用户要求旁路由在线时全屋去广告、旁路由宕机时不影响上网。
最终方案：DHCP option 6 只下发主路由自身 IP，客户端无法绕过 AdGuardHome。
dnsmasq server=10.10.10.2 + strictorder=1 + querytimeout=2s/retries=1，
旁路由在线时全部 DNS 经 AdGuardHome 去广告，宕机时 ~4s 超时后 fallback wan.dns。

## 清理 + 修复 `exit 0` bug

### Bug 5 — `exit 0` 导致 99-custom.sh 无法被 uci-defaults 自动删除
OpenWrt init 用 `. "$i" && rm -f "$i"` 执行 uci-defaults，source 的脚本中 `exit 0` 会终止父级 shell，
`rm -f` 永远不执行，脚本残留。修复：删除生成的 99-custom.sh 尾部 `exit 0`。

### HEADER 简化
`cat > "$OUT" <<'HEADER'` (3行) → `echo '#!/bin/sh' > "$OUT"` (1行)。

### DNS 全屋去广告确认
DHCP 只下发主路由 IP → 客户端只能走主路由 dnsmasq → `server=10.10.10.2` + `strictorder=1` +
`all-servers=0`(默认不并行) → 旁路由在线必走 AdGuardHome，不可绕过。宕机 ~4s fallback wan.dns 保底。
