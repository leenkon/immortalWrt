完整审查了项目所有核心文件，重点分析了 OAF（OpenAppFilter）替换实现。

### OAF 替换机制摘要
1. 仅主路由（*-main）类型才会询问是否安装 OAF
2. 在 `feeds install` 之前：删除 feeds 中旧的 oaf/luci-app-oaf/open-app-filter 包，并删除 package/OpenAppFilter
3. 从 GitHub clone 最新 destan19/OpenAppFilter 到 package/OpenAppFilter（绕过 feeds 版本）
4. 用本地 feature.cfg 替换仓库内默认规则文件
5. 用 oaf_files/app_icons/ 覆盖 luci-app-oaf 的图标资源目录
6. 向 .config 追加 `CONFIG_PACKAGE_luci-app-oaf=y`

### 发现的问题（见下方详细报告已告知用户）
- feature.cfg 中有两处重复 ID 条目（2017、2033、2080、3017、3026、3121）
- app_icons/ 命名基于数字 ID，与 feature.cfg 中的 ID 对应
- `MAIN_VER` 提取逻辑 (`${VERSION%.*}`) 对 25.12.0 → 25.12 正确，24.10.x → 24.10 正确
- `CFG_PREFIX` 解析逻辑（IFS split）对 "default-main" 会得到 prefix="default"，与 configs 文件名匹配
